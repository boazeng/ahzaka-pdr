מסמך PDR – אחזקה אורבנית

מסמך זה מאחד את כלל הדרישות העסקיות, הטכנולוגיות והתפעוליות של חברת אחזקה אורבנית למערכת דיגיטלית אחת כוללת.

מסמך מאוחד מפרק 1 + 2 (גרסת v1.0)

1. תקציר מנהלים (Executive Summary)

חברת אחזקה אורבנית מתמחה בהקמה, תפעול ותחזוקה של מתקני חניה רובוטיים ומכפילי חניה במבנים ציבוריים ופרטיים בישראל. המערכת המתוארת במסמך זה נועדה לרכז את כל פעילות האחזקה, השירות, ניהול הקריאות והידע המקצועי תחת פלטפורמה דיגיטלית אחת, חכמה ומבוססת בינה מלאכותית.

המערכת תאפשר ניהול קריאות שירות מקצה לקצה, הקצאת טכנאים חכמה, מאגר ידע מתקדם, אינטגרציה עם Priority ERP, ותקשורת מלאה עם לקוחות וטכנאים דרך WhatsApp ו‑Telegram.

יעדים עיקריים

  • שיפור זמני תגובה וזמני פתרון תקלות.
  • שקיפות מלאה מול הלקוח לגבי סטטוס הקריאות.
  • שיפור תכנון האחזקה המונעת והפחתת תקלות חוזרות.
  • הנגשת ידע מקצועי לטכנאים בזמן אמת.

2. תיאור החברה והשירותים

2.1 שירותי ליבה

  • תחזוקה שוטפת 24/7 למתקני חניה מסוגים שונים.
  • הקמה והתקנה של מתקני חניה רובוטיים ומכפילי חניה.
  • ניהול תפעולי, ניטור ודוחות ביצועים.
  • חיזוי תקלות והפחתת השבתות באמצעות AI.

2.2 סוגי מתקנים

  • מכפילי חניה ידניים.
  • מכפילי חניה חצי‑אוטומטיים.
  • חניונים רובוטיים מלאים.
  • מערכות Puzzle Parking מודולריות.

3. חזון, מטרות ו‑KPIs

3.1 חזון

להפוך את אחזקה אורבנית למובילה בישראל בתחום שירותי החניה החכמים, תוך מתן חוויית שירות מעולה, שקיפות מלאה, ויעילות תפעולית גבוהה המבוססת על טכנולוגיות בינה מלאכותית.

3.2 מטרות כמותיות

  • הפחתת זמן תגובה ממוצע לקריאות קריטיות מ‑4 שעות ל‑1.5 שעות.
  • הגעה לשביעות רצון לקוחות של 90% ומעלה.
  • עמידה ב‑SLA בשיעור 95% ומעלה.
  • גידול במספר הלקוחות הפעילים בשיעור 30% בשנה.

3.3 מטרות איכותיות

  • יצירת מאגר ידע ארגוני נגיש לכלל הצוותים.
  • שיפור רמת התיעוד והדיווח מכל קריאה.
  • הפחתת תלות בטכנאים ספציפיים בעלי ידע ייחודי.
  • יצירת תהליכי למידה מתמשכים מהשטח.

4. היקף הפרויקט

4.1 בתוך היקף (In Scope)

  • מערכת ניהול קריאות שירות (Ticketing) מלאה.
  • תקשורת דו‑כיוונית עם לקוחות וטכנאים ב‑WhatsApp ו‑Telegram.
  • הקצאת טכנאים חכמה לפי זמינות, מיומנויות ומיקום.
  • תיעוד דיגיטלי מלא לכל קריאה, כולל תמונות, וידאו וקבצים.
  • מערכת Knowledge Base עם חיפוש סמנטי.
  • דוחות ביצועים, דשבורדים ו‑KPIs.
  • מודול תחזוקה מונעת וחיזוי תקלות.
  • אינטגרציה עם Priority ERP.
  • ניהול SLA והתראות על חריגות.
  • ניהול הרשאות ואבטחת מידע.

4.2 מחוץ להיקף (Out of Scope)

  • שליטה ישירה במערכות PLC / SCADA של המתקנים.
  • אפליקציות Mobile Native (iOS/Android) – בשלב ראשון.
  • מערכת CRM מלאה לניהול לידים ומכירות.
  • מערכת נוכחות ושכר.
  • אתר תדמית ללקוחות סופיים.

5. ארכיטקטורה דיגיטלית

המערכת מבוססת על ארכיטקטורת Multi‑Agent, שבה מספר סוכני בינה מלאכותית ושרותי Backend עובדים יחד לתמיכה מלאה במחזור החיים של קריאות השירות והאחזקה.

5.1 שכבות המערכת

  • שכבת ממשק משתמש: WhatsApp Business API, Telegram Bot.
  • שכבת Gateway: שירות FastAPI לקבלת הודעות ו‑Webhooks.
  • שכבת תזמור: Autogen / LangChain לניהול שיחות בין סוכנים.
  • שכבת סוכנים: סוכני GPT/Claude לביצוע לוגיקה עסקית.
  • שכבת נתונים: PostgreSQL לנתונים מובנים, Qdrant לוקטורים וידע.
  • שכבת אינטגרציה: ממשקי Priority ERP ו‑REST APIs נוספים.

5.2 סוכנים מרכזיים

  • Gateway Agent – קבלת הודעות, זיהוי Intent וניתוב לסוכן המתאים.
  • Customer Service Agent – פתיחת קריאות ושירות לקוחות.
  • Technician Manager Agent – הקצאת טכנאים וניהול זמינות.
  • Knowledge Base Agent – השבת ידע טכני ומדריכים.
  • Fault Detection Agent – ניתוח דפוסים וחיזוי תקלות.
  • Reporting Agent – יצירת דוחות וניתוח ביצועים.
  • Priority Integration Agent – סנכרון עם Priority.
  • Notification Agent – שליחת התראות ועדכונים.

6. דרישות פונקציונליות (תקציר)

6.1 ניהול קריאות שירות

  • פתיחת קריאה חדשה דרך WhatsApp ע"י לקוח.
  • סיווג אוטומטי לפי חומרה ודחיפות.
  • מחזור חיים מלא של קריאה: new → assigned → in_progress → resolved → closed.
  • עדכוני סטטוס אוטומטיים ללקוח (הוקצה טכנאי, בדרך, הגיע, נסגר).
  • תיעוד קבצי מדיה וקבצים מצורפים לכל קריאה.

6.2 ניהול טכנאים

  • אלגוריתם הקצאה אוטומטית לפי מיקום, עומס, מיומנויות וסוג התקלה.
  • ניהול זמינות ומשמרות של טכנאים.
  • פרופיל טכנאי כולל מיומנויות, אזורי שירות ודירוג ביצועים.

6.3 מאגר ידע (Knowledge Base)

  • אחסון מדריכים, תרשימים, נהלים וסרטוני הדרכה.
  • חיפוש סמנטי באמצעות Qdrant.
  • תיעוד אוטומטי של פתרונות שהצליחו.

6.4 SLA ודוחות

  • הגדרת SLA לפי סוג לקוח, סוג מתקן ורמת דחיפות.
  • התראות על קריאות קרובות לחריגה מ‑SLA.
  • דשבורד בזמן אמת וסט דוחות אוטומטיים (יומי/שבועי/חודשי).

6.5 תחזוקה מונעת וחיזוי

  • לוח זמנים לתחזוקה מונעת לכל מתקן.
  • ניתוח היסטוריית תקלות וזיהוי דפוסים.
  • חיזוי רכיבים בסיכון גבוה להחלפה.

6.6 אינטגרציות חיצוניות

  • סנכרון דו‑כיווני עם Priority ERP (לקוחות, אתרים, חשבוניות ועוד).
  • שימוש ב‑WhatsApp Business API לתקשורת עם לקוחות.
  • תמיכה ב‑Telegram כערוץ גיבוי.

6.7 אבטחה והרשאות

  • אימות משתמשים לפי מספר טלפון והקצאת תפקיד.
  • מודל הרשאות RBAC (לקוח, טכנאי, ראש צוות, מנהל תפעול, מנכ"ל, כספים).
  • Audit Trail מלא לכל פעולה משמעותית במערכת.

7. דרישות לא‑פונקציונליות (תקציר)

7.1 ביצועים

  • זמן תגובה לפתיחת קריאה חדשה: פחות מ‑30 שניות.
  • זמן טעינת דשבורד ראשי: פחות מ‑2 שניות.
  • זמן חיפוש במאגר הידע: פחות מ‑1 שנייה.

7.2 זמינות ואמינות

  • זמינות מערכת (Uptime) שנתית: 99.5% ומעלה.
  • גיבויים אוטומטיים כל 6 שעות לכל רכיבי הנתונים.
  • RTO (זמן שחזור): עד 4 שעות.
  • RPO (אובדן נתונים מקסימלי): עד 6 שעות.

7.3 מדרגיות

  • תמיכה בעלייה פי 3‑5 במספר הלקוחות והקריאות תוך 3 שנים.
  • שימוש במסדי נתונים ותשתיות ניתנות להרחבה (PostgreSQL, Object Storage, Qdrant).

7.4 שימושיות

  • ממשק מבוסס WhatsApp – ללא צורך בהתקנת אפליקציה חדשה.
  • שפה טבעית ללקוחות ולטכנאים.
  • מינימום צעדים לפתיחת קריאה ועדכון סטטוס.

10. סטטוס אוטומציה

  • ניתן לייצא את תוכן המסמך לקובץ .md לצורך ניהול גרסאות.
  • ניתן לפרוס את קובץ ה‑HTML על גבי שרת סטטי (למשל S3 / GitHub Pages).
  • בעתיד ניתן יהיה לעדכן סעיפים ספציפיים באופן אוטומטי ע"י סוכן AI.
